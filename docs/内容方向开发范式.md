# 内容方向开发范式

> conv-edit 添加新内容方向（新游戏/新视频类型）的标准流程。
> 核心理念：**先理解内容，再写代码。**

---

## 范式流程

```
① 游戏知识文件 → ② 参考视频分析 → ③ Preset 配置 → ④ 特效素材 → ⑤ 验证迭代
```

### ① 创建游戏/内容知识文件

路径：`presets/{游戏名}-游戏理解.md`

必须覆盖：

| 章节 | 内容 | 为什么需要 |
|------|------|-----------|
| 游戏基础 | 类型、模式、平台 | 理解素材是什么 |
| 核心术语 | 黑话、社区梗 | 配字不能外行 |
| 标准流程 | 一局游戏的阶段顺序 | 确定叙事弧线 |
| 内容生态 | 抖音/B站热门风格 | 知道观众爱看什么 |
| 剪辑指南 | 素材优先级、节奏建议 | 决定切点密度 |
| 后期特效 | 该用什么特效、什么时候用 | 不做无关特效 |
| 踩坑清单 | 外行常犯错误 | 避免低级错误 |

**原则：**
- 写这个文件的人必须玩过或深入研究过这个游戏
- ❌ 不要根据通用 FPS 知识猜测 — 每个游戏的社区文化完全不同
- ✅ 看至少 5-10 个该游戏的热门视频，提取共性

### ② 分析参考视频

拿一个该方向的标杆视频做完整拆解：

1. **下载视频** → 提取帧 + 音频
2. **场景检测** → 分析切点节奏
3. **VLM 帧分析** → 理解每个时间段在干什么
4. **音频分析** → 语音/BGM/音效比例
5. **输出分析报告** → 结构、手法、风格参数

### ③ 配置 Preset

路径：`presets/{方向名}.yaml`

基于知识文件和视频分析结果，配置：

```yaml
name: 方向标识
game: 关联游戏名
game_knowledge: presets/{游戏名}-游戏理解.md  # 引用知识文件

scene_detection: ...    # 基于素材类型调参
segments: ...           # 片段长度偏好
narrative_arc: ...      # 基于游戏流程设置弧线
render:
  keep_ui: true/false   # 是否保留游戏UI
  vocal_reduction: ...  # 是否去人声
overlays: ...           # 默认叠加层
effects: ...            # 专属特效列表
```

**关键参数的决策依据：**

| 参数 | 问自己 |
|------|--------|
| `vocal_reduction` | 这个游戏的真人语音重要吗？ |
| `keep_ui` | 观众需要看血量/地图/弹药吗？ |
| `beat_sync` | 语音节奏 vs BGM节奏，哪个优先？ |
| `scale_mode` | 留黑边（保留信息）vs 裁满屏（视觉冲击）？ |
| `max_duration` | 观众需要几秒看懂一个片段？ |

### ④ 准备特效素材

路径：`effects/{游戏名}/`

每个特效需要：
- 素材文件（PNG/MP4）
- 触发时机说明（什么游戏事件触发）
- 参数（位置、大小、持续时间、透明度）

特效应该**精准回应游戏事件**，不是随机加的：
- ✅ "击杀时枪口发光" — 有明确的触发条件
- ❌ "每 5 秒闪一下" — 跟游戏内容无关

### ⑤ 验证迭代

1. 用真实素材跑渲染
2. VLM 自检效果（字幕是否匹配画面、字体大小是否合适）
3. 人工看一遍
4. 对比参考视频，差距在哪？
5. 回到步骤 ③ 调整参数

## 通用 vs 专属边界

| 层 | 放什么 | 例子 |
|----|--------|------|
| 🔵 通用渲染管线 | 切分/缩放/混音/叠加 | `overlay.py`, `build_clip_vf()` |
| 🔵 通用剪辑参数 | 场景检测/弧线/长度 | `scene_detection` 字段 |
| 🟠 游戏专属 | 术语/特效/字幕风格 | `三角洲行动-游戏理解.md` |
| 🟠 方向专属 | preset YAML | `douyin-comedy.yaml` |

**原则：一个参数如果能被多个游戏/方向复用，就放通用层。如果是某个游戏的独有知识，放专属文件。**

## 反模式（不要做的事）

| ❌ | ✅ |
|----|-----|
| 没建知识文件就开始写 preset | 先建知识文件，确认理解了对再写 preset |
| 用通用 FPS 知识猜测三角洲 | 搜该游戏的热门视频，了解真实社区 |
| AI 编对话当字幕 | 保留原声，字幕只标注 |
| 特效随便加 | 每个特效必须有触发条件和游戏关联 |
| 所有方向用同一套参数 | 不同游戏/方向必须独立调参 |
