# Conv-Edit 安装与使用指南

> 对话式 AI 视频剪辑工作台。丢视频 → 自动分析 → 聊天驱动剪辑 → 渲染下载。

---

## 1. 环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.10+ | |
| ffmpeg | 4.0+ | 视频分析/渲染核心，PATH 中可用 |
| 磁盘空间 | ≥2GB 空闲 | 渲染产物临时存储 |

### LLM（可选，智能选片/审查需要）

| 模型 | 用途 | 说明 |
|------|------|------|
| 文本模型 | 时间线审查、风格推荐 | 如 `qwen3.6:27b`（Ollama）或任何 OpenAI 兼容 API |
| 视觉模型 | 智能选片、画面审查 | 如 `minicpm-v:8b`（Ollama） |

不配置 LLM 也可以使用手动模式（纯算法选片 + 风格预设）。

---

## 2. 安装

```bash
# 克隆
git clone https://github.com/Gousting/conv-edit.git
cd conv-edit

# 安装依赖
pip install fastapi uvicorn librosa scipy ruptures httpx

# 验证 ffmpeg
ffmpeg -version
```

---

## 3. 启动

```bash
python3 server.py
```

浏览器打开 **http://localhost:8765**

```
✓ 服务启动在 http://0.0.0.0:8765
```

## 5. 三条使用路线

### 5.1 手动模式（零 LLM，全程算法）

```
1. 丢视频到左侧 → 自动切分+标注（镜头检测/强度/音频模式）
2. 选风格预设（右侧 🎬）
3. 手动勾选/取消片段
4. （可选）上传 BGM → 自动分析节拍+波形
5. 点 🎯 生成时间线 → 查看热力图
6. 点 🚀 渲染 → 等待完成 → 点 📥 下载
```

全程不调任何 API，适合精确控制。

### 5.2 半自动模式（💬 聊天引导）

```
1. 丢视频 → 聊天面板自动弹出
2. AI: "这是什么内容？想做什么风格？"
3. 你: "永劫无间击杀集锦，快节奏燃的"
4. AI: 自动设风格 + 智能选片 → "已选 12 个片段，行吗？"
5. 你: "第3个不要" / "再加两个慢动作"
6. AI: 调整 → 提示上传 BGM → 生成 → 审查 → 渲染
```

像跟剪辑师聊天，不需要懂参数。

### 5.3 全自动模式（💬 一句话）

```
1. 丢视频
2. 丢 BGM
3. 在聊天里说: "帮我自动规划实现"

AI 一口气完成：
  智能选片 → 生成时间线 → LLM 审查 → VLM 审查 → 渲染 → 下载链接
```

## 6. 聊天指令速查

| 你说 | AI 做 |
|------|------|
| "这是永劫无间击杀集锦" | 分析内容 → 推风格 → 选片 → 等你确认 |
| "快节奏燃的" | 切换风格预设 |
| "第3个不要" | 取消片段 3 |
| "加两个慢动作" | 额外选取慢动作片段 |
| "上传 BGM" | 提示你拖入 BGM 文件 |
| **"帮我自动规划实现"** | 选片→生成→审查→渲染→下载，全部自动 |
| "重新选片" | 清空重新来 |
| "审查结果怎么样" | 展示 LLM/VLM 审查发现的问题 |

## 7. 剪辑风格预设

| 预设 | 策略 | 强度 | 音频处理 | 选片偏好 |
|------|------|------|----------|----------|
| 🎮 游戏集锦 | fit | ≥0.7 | 纯 BGM | 短片段(<4s) + 高强度 |
| 🎬 电影对白 | fit | ≥0.3 | 突出人声 | 中长(3-10s) + 有对话 |
| 🏔️ 旅行 Vlog | fit | ≥0.5 | 融入 BGM | 高低强度交替 |
| 🎵 MV 踩点 | loop | ≥0.8 | 纯 BGM | 极短 + 全选 |
| 📖 叙事短片 | fit | ≥0.2 | 融入 BGM | 保留原顺序 |
| 🤖 智能推荐 | fit | ≥0.6 | LLM 决定 | LLM 分析后推荐 |

| 字段 | 说明 | 示例值 |

| API 地址 | OpenAI 兼容接口 | `http://localhost:11434/v1` |
| API Key | 可选（Ollama 不需要） | — |
| 文本模型 | 审查时间线用 | `qwen3.6:27b` |

支持 Ollama、vLLM、OpenAI、SiliconFlow 等所有 OpenAI 兼容接口。

## 10. 核心算法

| 步骤 | 算法 |
|------|------|
| 镜头检测 | ffmpeg `scdet` 滤镜 |
| 强度评分 | ffprobe `volumedetect` → RMS → 0~1 归一化 |
| 音频分类 | ffmpeg `silencedetect` → 人声/音乐/静音 |
| BPM/节拍 | librosa `beat_track` |
| 切点规划 | 强度匹配 + 节拍吸附 + 二次裁剪 + protect_end |
| 变速补偿 | 仅纯 BGM 片段(±5%)，人声片段禁止变速 |
| 间隙修复 | <500ms 拉伸 / 0.5-2s 速度波纹 / ≥2s 拒绝 |

---

## 11. 约束保护

- **防爆音** — `jcut_audio_lead_ms` ≥ `gain_fade_ms`
- **人声保护** — 含人声片段 · 变速 → 报错
- **能量保护** — `protect_end` 确保高能画面不落入低能尾奏
- **循环上限** — BGM 最多循环 3 次
- **音频归一化** — 全部重采样 48kHz/stereo

## 12. 常见问题

**Q: 没有 LLM 能用吗？**
A: 可以。手动模式全程纯算法，不调任何 API。

**Q: 为什么智能选片结果不理想？**
A: 视觉模型质量决定了选片效果。推荐 `minicpm-v:8b` 或更强模型。选片不满意可以说"第X个不要"手动调整。

**Q: 渲染很慢？**
A: 取决于片段数量和分辨率。片段越多、分辨率越高越慢。可以先降低分辨率快速预览。

**Q: 支持什么视频格式？**
A: ffmpeg 支持的所有格式（mp4/mkv/mov/avi 等），推荐 H.264 编码的 mp4。

**Q: 输出文件在哪？**
A: `outputs/` 目录下的 `renders/` 子目录，浏览器下载按钮可直接下载。
